## 怎么看这个仓库

建议按下面顺序使用：

1. 先看你当前正在使用的编辑器/代理目录
2. 再读取该目录下对应的规则入口文件
3. 根 `README.md` 只负责导航和状态说明，不重复展开具体规则细节

## 当前已沉淀的核心规则

目前最完整的规则来源于 `codex/AGENTS.md`。根文档只保留摘要，详细内容请直接查看原文件。

已明确的高层约束包括：

- **工作方式**：先收集证据，再规划，再实施，避免无依据假设
- **语言输出**：默认使用简体中文，复杂任务强调计划、进度、风险和验证结果
- **调试策略**：不靠静默降级、假成功路径或隐藏错误来“先跑起来”
- **质量基线**：强调小函数、小文件、低嵌套、可测试、少魔法数字
- **安全要求**：不硬编码密钥，不拼接危险输入，边界输入要校验
- **危险操作**：删除、批量修改、环境变更、数据变更、生产调用等需要先确认

## 适合放什么内容

当前仓库更适合按“编辑器 / 代理”维度沉淀规则，例如：

- 回复语言
- 工具调用偏好
- 规划与执行流程
- 是否需要先确认再动手
- 提交信息、PR 描述、输出格式约束

如果后续要扩展技术栈规则，建议在新增目录后再写入根文档，避免把计划中的结构写成已存在内容。

## 推荐维护方式

为避免文档互相复制、后续难维护，建议遵守下面几条：

- 根 `README.md` 只做导航和状态说明
- 每个编辑器目录只描述该工具自己的行为约束
- 同一条规则尽量只在一个地方维护，其他文档通过引用指向它
- 新增目录前，先确认该目录已经实际创建，再同步更新根文档
